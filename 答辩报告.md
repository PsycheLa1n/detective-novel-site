# 推理小说检索网站 — 设计与实现

## 摘要

本项目构建了一个面向推理小说爱好者的垂直检索平台，支持关键词搜索、多标签筛选、年份范围过滤、作者筛选和多维度排序功能。网站收录119本经典推理小说，涵盖60余位作家和33个分类标签。采用Node.js + Express后端、SQLite数据库、原生前端技术栈，实现了基于SSE（Server-Sent Events）的实时留言板和用户登录系统，并通过内网穿透技术实现公网访问。

**关键词**：推理小说检索、SSE实时推送、SQLite、内网穿透、SVG动态封面

---

## 一、项目背景与目标

### 1.1 项目背景

推理小说是全球最受欢迎的文学类型之一，拥有庞大的读者群体。然而，市面上缺乏专门面向推理小说的垂直检索工具——普通书店和读书网站通常将推理小说笼统归类为"悬疑"，无法满足读者按流派（本格/社会派/新本格）、诡计类型（密室杀人/建筑诡计/叙述性诡计）、题材风格等维度进行精细化检索的需求。

### 1.2 项目目标

1. 构建完整的推理小说数据库，收录100本以上经典作品
2. 提供多维度筛选与排序功能，支持标签、年份、作者、关键词的组合查询
3. 实现用户登录与实时留言板功能
4. 纯前端生成书籍封面，零外部图片依赖
5. 支持公网访问，无需云服务器部署

### 3.2 核心SQL设计

**书籍表（books）**：存储119本小说的元数据，包括书名、作者、出版年份、剧情简介和评分。

## 四、核心功能实现

### 4.1 多维度搜索与筛选

搜索/筛选的全部逻辑集中在 `/api/books` 这一个API端点中，通过动态SQL拼接实现。

**支持的筛选维度**：

| 参数 | 说明 | 示例 |
|------|------|------|
| `q` | 关键词模糊匹配书名和作者 | `?q=东野` → 返回3本东野圭吾作品 |
| `tags` | 标签ID列表，逗号分隔，交集逻辑 | `?tags=1,5` → 同时包含"本格推理"和"密室杀人" |
| `yearFrom` / `yearTo` | 年份范围过滤 | `?yearFrom=1990&yearTo=2010` |
| `author` | 作者名模糊匹配 | `?author=阿加莎` |
| `sort` | 排序字段 | `rating_desc` / `year_asc` / `title_asc` 等6种 |

**实现原理**：

```sql
-- 后端根据参数动态拼接的SQL示例
SELECT DISTINCT b.* FROM books b
INNER JOIN book_tags bt ON b.id = bt.book_id
WHERE b.title LIKE '%密室%'
  AND bt.tag_id IN (1, 5, 12)
  AND b.year >= 1990 AND b.year <= 2020
ORDER BY b.rating DESC
LIMIT 50
```

前端使用300ms防抖机制，每次用户修改筛选条件后自动触发API请求，实现即时反馈。

### 4.2 实时留言板（SSE）

留言板采用Server-Sent Events技术，实现新留言的实时推送。

**工作流程**：

```
用户A发送留言
  │
  ▼
POST /api/messages → 存入数据库 → 服务器遍历所有SSE连接
                                      │
                                      ▼
                              推送JSON数据给每个连接的浏览器
                                      │
                                      ▼
                              用户B/C/D的页面即时显示新留言
```

**为什么选择SSE而非WebSocket**：
- SSE是HTTP标准的一部分，无需额外协议升级
- 留言板只需服务器→客户端的单向推送，SSE完全满足需求
- 浏览器原生支持`EventSource` API，自动重连
- 比WebSocket更节省资源

**客户端实现**：
```javascript
const eventSource = new EventSource('/api/messages/stream');
eventSource.onmessage = (event) => {
    const msg = JSON.parse(event.data);
    prependMessage(msg);  // 插入到留言列表顶部，带高亮动画
};
```

### 4.3 SVG动态封面生成

网站所有书籍封面均通过JavaScript在前端实时生成，无需任何图片资源文件。

**实现原理**：
1. 根据书籍ID从25种渐变配色方案中选择一种
2. 动态生成SVG XML字符串：包含渐变色背景、装饰线条、圆形装饰、书名和作者名
3. 将SVG转为Base64编码
4. 作为Data URI嵌入`<img>`标签的`src`属性

**代码示例**：
```javascript
function generateBookCover(bookId, title, author) {
    const palette = COVER_PALETTES[bookId % 25];
    const svg = `<svg xmlns="..." width="400" height="560">
        <defs>
            <linearGradient id="bg">
                <stop offset="0%" stop-color="${palette[0]}"/>
                <stop offset="100%" stop-color="${palette[1]}"/>
            </linearGradient>
        </defs>
        <rect width="400" height="560" fill="url(#bg)"/>
        <text x="200" y="240">${title}</text>
        <text x="200" y="470">${author}</text>
    </svg>`;
    return 'data:image/svg+xml;base64,' + btoa(unescape(encodeURIComponent(svg)));
}
```

**优势**：
- 零外部资源依赖，不增加HTTP请求
- 每本书独一无二的配色
- SVG矢量图在任何分辨率下清晰显示

### 4.4 轮播图

首页轮播展示评分最高的8本小说，采用纯CSS + JavaScript实现：
- `setInterval`每4秒自动切换到下一张
- 鼠标悬停暂停自动播放，移开恢复
- 点击左右箭头或底部指示器手动切换
- `transition: transform 0.5s cubic-bezier(0.4, 0, 0.2, 1)`实现平滑滑动

### 4.5 用户登录与会话管理

- 默认账号：`user / password`
- 使用`express-session`管理会话，cookie有效期24小时
- 登录状态影响：留言板输入框（登录后可发言）、导航栏显示（用户名+退出按钮）
- 密码以明文存储于SQLite中（该项目定位为个人演示项目，非生产环境）

### 6.2 公网访问

通过ngrok内网穿透技术，将本地`localhost:3000`暴露为公网HTTPS地址：

```bash
ngrok http 3000
# 生成 https://xxx.ngrok-free.dev 公网地址
```

**内网穿透原理**：ngrok客户端与ngrok云服务器之间建立一条加密的TCP隧道。外部用户访问ngrok生成的公网域名时，请求被ngrok服务器接收，然后通过隧道转发到本地3000端口，响应再原路返回。

**优势**：
- 无需购买云服务器和域名
- 无需备案
- 自动HTTPS加密
- 一行命令即可启动
